Sobre STOP:
- Se turns_used < min_turns: "stop" DEVE ser false (a reunião precisa de mais debate).
- Se open_questions_count > 0: "stop" DEVE ser false (há perguntas por responder).
- Se last_speaker_maturity < 4: evite stop (ainda não há maturidade suficiente).
- Só use "stop": true quando: turns_used >= min_turns E open_questions_count == 0 E maturidade geral >= 4.

Sobre NEXT_SPEAKER:
- Se há perguntas abertas, escolha o agente que melhor pode respondê-las.
- Se um agente falou menos que os outros (ver distribuição de falas), priorize-o.
- Evite escolher o mesmo orador 3 vezes seguidas sem justificação forte.
- Se estiver perto do max_turns, priorize o agente de Execução para fechar o plano.

Sobre REASON:
- Seja ESPECÍFICO: diga exactamente O QUE falta resolver ou quem precisa responder a quê.
- Exemplos BOM:  "Agente B levantou risco X que Agente C ainda não respondeu"
- Exemplos MAU: "A proposta precisa de mais análise" (vago demais)

Sobre AGENDA_UPDATE:
- Se durante o debate surgiu um ponto novo não coberto pela agenda, acrescente-o.
- Se um item da agenda já foi resolvido, pode indicar isso na reason.
